# Antigravity Portfolio: Agentic AI Engineering

This repository is a showcase of high-fidelity projects architected and built entirely using **Antigravity**, an agentic AI coding environment. It demonstrates the shift from simple AI prompting to robust AI engineering workflows.

## 🚀 Overview
The projects in this repository were built through iterative collaboration with an AI agent. Each folder represents a unique challenge solved by combining AI reasoning with standard software engineering practices (HTML/CSS, Python, Node.js).

## 📁 Projects

### 1. [Hybrid AI Image & Carousel Engine](./image-generation/)
An automated design-to-code pipeline for generating professional social media assets.
- **Problem**: AI-generated images often lack brand consistency and typographic precision.
- **Key Tech**: Node.js, Puppeteer, HTML/CSS, Imagen/DALL-E.

#### 🚧 Challenges & Overcoming Roadblocks
- **Moving Beyond Generic Tools**: I initially tried using Stitch for carousel generation, but found it lacked the precision needed for a high-end brand. I overcame this by developing a custom **HTML/CSS/Puppeteer pipeline**, giving me 100% control over the output.
- **Solving "Reference Fatigue"**: Early on, I was referencing too many images, which confused the AI and led to inconsistent designs. I fixed this by narrowing down to a single "Master Reference" and distilling all visual rules into a comprehensive **Design Skill file**, ensuring every generation followed one verified DNA.

### 2. [SEO_GEO Content Pipeline](./seo-geo/)
A multi-agent system for producing human-sounding, search-engine-optimized, and generative-engine-optimized (GEO) blog content.
- **Problem**: AI content often sounds robotic and fails to cite sources or optimize for new "AI Search" patterns.
- **Key Tech**: Python, Google Docs API, Multi-Agent Logic.

#### 🚧 Challenges & Overcoming Roadblocks
- **The Google Integration Hurdle**: A major roadblock was figuring out how to push the final content directly to Google Docs. I solved this by implementing a custom **Python automation script** using the Google Docs API.
- **Environment Barriers**: I faced issues with local setup, as Python and the required libraries weren't initially installed on my machine. I overcame this by setting up the entire development environment from scratch to support the automation pipeline.

Building these projects required more than just "generating code." It involved:
