# LinkedIn Post Writing

## When to use this skill
- The user already has a **Post Brief/Framework** (generated by `designing-linkedin-frameworks`) and wants full posts.
- The user provides a strategy/plan from `planning-linkedin-content` and wants to skip straight to drafting.
- The user shares raw notes or bullet points and wants them turned into LinkedIn posts.
- The user provides a draft and wants a cleaner, higher-performing version.
- The user wants repeatable post patterns based on a consistent structure.

## Inputs this skill expects
- Post Brief / Framework (Goal, ICP, Angle, Triggers, Format, CTA).
- Strategic Context (Pillar, Cluster, or Branch from `planning-linkedin-content`).
- **Refer to `knowledge_company/brand/brand_voice.md`** for tone guidelines.
- **Consult `knowledge_company/product/features.md`** for technical/value specifics.
- One or more selected hooks (optional but ideal).
- Raw content: notes, stories, data, opinions, or lessons.
- Tone preference (e.g., direct, conversational, analytical, vulnerable).
- Any constraints (e.g., no hard selling, avoid certain phrases).

If the user only provides minimal input, derive a simple internal framework and still produce a post.

## Default post structure
Unless the user requests otherwise, use:

1. Hook (1–3 lines)
2. Context or tension (what’s wrong, surprising, or overlooked)
3. Insight, story, or framework (the “meat”)
4. Specifics (details, examples, numbers, micro-stories)
5. Takeaway (clear learning or message)
6. CTA aligned to goal

## Workflow

### PLAN: Build a mini-brief
Summarize inputs internally:

```text
Goal: [comments / reach / saves / leads / profile visits]
ICP: [who this is for]
Angle: [what the post wants to say]
Trigger(s): [primary + secondary]
Hook(s): [selected hook or to be generated]
Tone: [e.g., calm authority, sharp opinion, warm story]
Length: [short / medium / long]
Format notes: [text-only / text + asset]
CTA: [desired behavior]
```

### VALIDATE: Quick alignment checks
- Does the angle match the hook?
- Is the post promising something specific?
- Is there at least one concrete example, detail, or step to include?
- Does the goal suggest a particular CTA type?

If not, adjust the angle or ask one clarifying question.

### EXECUTE: Write the post

**0. Inject Brand & Product Context**
Before writing, consult the `knowledge_company/` to ensure accuracy:
- **Brand Personality**: Align with `knowledge_company/brand/brand_voice.md`.
- **Product Proof**: Use features/data from `knowledge_company/product/features.md`.
- **Value Differentiators**: Align with `knowledge_company/positioning/value_propositions.md`.

#### 1. Place or generate the hook
- If a hook is provided, use it as-is or lightly refine.
- If not, generate one hook based on the mini-brief logic.

#### 2. Expand the body
Depending on content type:

- Story:
  - Situation → conflict → decision → result → lesson.
  - Keep paragraphs short, with 1–3 sentences per line.

- Framework or checklist:
  - Brief setup → numbered or bulleted points → short explanation for each.
  - Make each point specific and actionable.

- Opinion:
  - Clear stance → rationale → example → implication for the ICP.

- Case study:
  - Before → intervention → after → meta-lesson.

#### 3. Align to LinkedIn reading patterns
- Short paragraphs and line breaks.
- Avoid heavy formatting or long, dense blocks.
- Make it skim-friendly but still meaningful.

#### 4. Add CTA
Match CTA to goal:

- Comments:
  - Ask for specific experiences, examples, or disagreement.
- Reach:
  - Use share-worthy framing and a light question.
- Saves:
  - Suggest “Save this for later” when the post is a framework or checklist.
- Leads:
  - Use soft invitation: “If you want help doing this, send a DM.”

### Output format
Return:

1. Strategy summary (short)
```text
Goal: ...
ICP: ...
Angle: ...
Tone: ...
CTA: ...
```

2. LinkedIn post (ready to copy-paste)

3. Optional variant
- Same angle, different hook, or different CTA intensity.

## Draft improvement mode
When the user provides a post:

- Identify issues:
  - Slow or generic hook.
  - Unclear ICP.
  - Overly abstract content.
  - Weak or misaligned CTA.
  - Wall-of-text formatting.

- Rewrite:
  - Sharper hook.
  - Clearer ICP orientation.
  - More specific examples.
  - Cleaner flow.
  - Better mobile formatting.

- Explain:
  - Briefly state what changed and why.

## Quality checklist
Before finalizing:

- The post is clearly for one ICP.
- The hook aligns with the body and feels non-generic.
- The post includes at least one concrete, memorable detail.
- Paragraphs are short enough for mobile.
- The CTA matches the stated goal.
- Tone matches what the user requested.
- The post reads naturally and not like an ad or a blog pasted into LinkedIn.
